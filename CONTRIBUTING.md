# Contributing

Thanks for helping improve this skills repository.

## Ground rules

- Keep each skill focused on a single job.
- Prefer concrete, operational wording over abstract advice.
- Avoid hidden assumptions; document prerequisites explicitly.
- Preserve frontmatter shape (`name`, `description`) in each `SKILL.md`.

## Contribution workflow

1. Fork and create a feature branch.
2. Add or update skill files.
3. Run a self-review against the quality checklist in `README.md`.
4. Open a PR with:
   - Problem statement.
   - Trigger phrases.
   - Before/after behavior summary.

## PR quality checklist

- [ ] Skill has clear trigger phrases.
- [ ] Workflow is stepwise and deterministic.
- [ ] Output format is specified.
- [ ] Non-goals are explicit.
- [ ] Links/references are valid.
- [ ] Docs are typo-checked.

## Style guidance

- Use short headings and imperative verbs.
- Keep examples realistic and concise.
- Prefer bullets over long paragraphs for operator instructions.

## Code of conduct

By participating, you agree to follow [`CODE_OF_CONDUCT.md`](./CODE_OF_CONDUCT.md).
